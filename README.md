# LibAR — 서가 스캔 데모 (2026 도서관 데이터 활용 공모전)

스마트폰/PC 브라우저에서 도서관 서가를 스캔해 청구기호를 인식하고 오배열을 찾아내는
**온디바이스 AI** 데모입니다. 검출→인식→장서 대조→오배열 판정까지 전 과정이 브라우저 안에서
이루어지며 사진은 서버로 전송되지 않습니다.

**데모:** https://ainsof.dev/libar-demo/

## 기능
- **서가 점검하기**: 카메라를 비추면 실시간 라벨 검출 → 화면이 안정되면 자동 판독(일시정지 불필요),
  프레임마다 결과를 누적(투표)해 🟢확인 / 🔴잘못 꽂힘 / ⚪다시 확인으로 판정
- **도서 찾기**: 종합자료실 전 장서 **20,944권** 검색 → 층별 실배치도 길안내 → 서가 스캔에서 찾는 책 📍핀 강조
- **샘플 서가**: 촬영해 둔 대림도서관 서가 2장으로 전체 흐름 바로 체험
- **학습용 조각 제공(선택)**: 동의하면 라벨 조각만 팀 드라이브로 전송해 모델 개선에 사용
- **수집 모드(팀용, `#collect`)**: 인식을 생략하고 검출→조각→전송만 — 사진당 3~5초, 라이브 4초 자동 수집
- **결과 리포트**: 판독 화면(주석)+판정 목록을 한 장 이미지로 합성해 팀 드라이브로 전송 — 점검 기록 보존
- **고해상 원샷 판독**: 라이브 중 자동 판독이 발동하는 순간만 카메라 최대 해상도로 한 컷 확보 — 걷기 UX·발열은 그대로, 라벨 픽셀만 커짐

## 온디바이스 모델
| 단계 | 모델 | 크기 |
| :-- | :-- | :-- |
| 라벨 검출 | YOLO26n (자가 학습 v3, 동적 입력 — 라이브 960px/사진 1280px) | 11MB |
| 텍스트 줄 검출 | PP-OCRv5 mobile det | 4.5MB |
| 청구기호 인식 | 한국어 rec, 저해상 라벨 파인튜닝 v4 (fp16) | 48MB |

ONNX Runtime Web — WebGPU 우선, 미지원 기기는 WASM 자동 폴백 (판독 결과 동일).
첫 실행 시 모델을 내려받아 이후엔 캐시로 동작합니다. 발열 억제를 위해 검출 페이싱·버퍼 재사용을
적용했고, 가로모드 회전 시 카메라를 자동 재획득합니다.

실증: 영등포구립 대림도서관 · 장서 정보는 공개 서지 데이터(청구기호·서명·구간·자료실)입니다.
